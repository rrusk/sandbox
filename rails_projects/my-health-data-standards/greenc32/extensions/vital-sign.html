3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

The Vital Sign class is a subset of the Results class but reported separately to follow clinical convention.

4.2. Description

The Vital Sign class, a subset of Results, is composed of the same attributes as Results. The attributes for the Results class are characterized in Results Extension Description. The Results attributes are also populated for convenience in the table below.

4.3. Clinical Requirement

Vital Sign, a subset of Result, is used to capture information about:

...[C]urrent and relevant historical vital signs for the patient. Vital Signs are a subset of Results (see section 2.1.2.14), but are reported in this section to follow clinical conventions. The differentiation between Vital Signs and Results varies by clinical context. Common examples of vital signs include temperature, height, weight, blood pressure, etc. However, some clinical contexts may alter these common vitals, for example in neonatology "height" may be replaced by "crown-to-rump" measurement. HITSP C/83
