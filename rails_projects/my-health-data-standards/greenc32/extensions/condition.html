3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Condition contains information about the problems and conditions associated with a patient. Condition characterizes the central reason a Provider seeks a Referral Consult Request to another Provider. As such, Referral Consult Request depends on Condition. However, Condition is not dependent upon Referral Consult Request and is used in many other subject classes.

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Unique id for the condition
code cd O The coded value of the condition represented
status string O

The status of the problem. Will contain the strings: active, inactive or resolved.

The status strings map to the following SNOMED CT codes:

  • 55561003 - Active
  • 73425007 - Inactive
  • 413322009 - Resolved
name string M This is a text description of the problem or condition.
effectiveTime TimeInterval O This is the range of time of which the condition was active for the patient or subject.
causeOfDeath boolean O Indicates that this problem was one of the causes of death for the patient or subject of the condition.
type cd O

This is a fixed value indicating the level of medical judgment used to determine the existence of a problem. Valid values from C80/2.2.3.1.2 are as follows:

  • 404684003
  • 418799008
  • 55607006
  • 409586006
  • 64572001
  • 282291009
  • 248536006
treatingProvider URL O Link for the person or organization performed or hosted the Procedure
dateTimeOfDeath dateTime O The date/time the patient or subject's death occurred
diagnosisPriority integer O

A number indicating the significance or the priority of the diagnosis code. It is used to distinguish between the primary and other diagnoses

  • 0 - Not included in diagnosis ranking
  • 1 - The primary diagnosis
  • 2 - For ranked secondary diagnoses

4.3. Clinical Requirement

Condition is characterized by:

  • name of the problem
  • duration of problem
  • level of medical judgment used to determine the existence of the problem
  • the significance or priority of the diagnosis code
  • the resolution status of the problem
  • other information necessary for a consulting provider to understand the purpose of the Referral Consult Request

4.4. XML Schema Definition (XSD)

The XML Schema Definition (XSD) for Condition specifies how to formally describe the elements in an XML document. For the REST+ Pilot, C32/C80 values that lend themselves to greater accuracy through calculation are calculated, including "AgeAtOnset" and "AgeAtDeath.".

Condition.xsd is located at the following URI: https://raw.github.com/ssayer/health-data-standards/develop/greenc32/schemas/rest_plus/condition.xsd

Condition.xsd extends entry which is defined in the data_types schema at the following URI: https://raw.github.com/ssayer/health-data-standards/develop/greenc32/schemas/rest_plus/data_types.xsd

4.5. Resource Representations

Conditions may be represented in XML and JSON format.
