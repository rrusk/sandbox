3. Extension Background

All hData section documents identified by an XML namespace must be declared in the extensions node of the root document. Extensions are either "mandatory" or "optional." Each section document has an address expressed as a Uniform Resource Identifier (URI). Each URI is registered in the extensions node of the root document. Sections can contain either single data type documents (e.g., XML, JSON, HTML) or URIs referencing other sections. Section document content must be expressed in XML and document types must be declared in the extensions node of the root document.

4. Description

4.1. Purpose

Procedure contains information defining the entry describing a procedure performed on a patient. 0 to many procedures can be associated with a patient.

4.2. Description

NameTypeMandatory/OptionalNotes
id string M Unique id for the procedure
code cd O The coded value of the procedure represented
status string O

The status of the procedure.

The status strings map to the following:

  • Aborted - Not Completed
  • Active - In Progress
  • Held - On Hold
  • Cancelled - Cancelled
  • Completed - Completed
Description string M Free text describing the Procedure.
effectiveTime TimeInterval O The date and time of the Procedure, including duration if pertinent.

performer URL O Link for the person or organization performed or hosted the Procedure

4.3. Clinical Requirement

Procedure is characterized by:

  • Procedure description;
  • Anatomical site of procedure;
  • Healthcare provider who performed the procedure; and,
  • Procedure identifier, code, status, time/date/duration, location, duration.
